# 2025-HuaQiFrontend

本项目为2025花旗杯金融创新应用大赛第一道赛题数据驱动的外汇风险信号模型前端项目，基于 Vue 3 + Vite + TypeScript 构建，包含实时汇率、汇率预测、回测分析与可解释性展示等模块。

## 项目背景

在外汇场景中，用户通常同时面临三类问题：

- 数据分散：实时行情、历史走势、策略回测结果来源不同，难以统一查看。
- 决策成本高：从“看行情”到“形成判断”需要大量人工整理和比对。
- 结果不透明：仅有预测结论但缺少解释，难以建立使用信任。

本项目以“可视化 + 预测 + 可解释”为核心设计思路，提供一个面向教学、研究与演示的前端平台，将实时汇率、策略回测、模型预测与解释报告整合到同一套交互流程中。

## 项目目标

- 构建可复用的外汇分析前端框架。
- 降低从数据观察到策略比较的操作门槛。
- 通过解释性模块提升模型结果可理解性。
- 为后端模型服务提供清晰、可扩展的前端接入层。

## 功能概览

- 实时汇率看板：展示主要货币对实时数据与趋势。
- 汇率预测：展示模型预测结果、解释文本与分析报告下载。
- 策略回测：展示 K 线、预测线、净值曲线与收益指标。
- 用户模块：登录、注册、密码修改、退出登录。

## 核心亮点

- 一体化分析流：支持从实时行情快速跳转到预测与回测模块。
- 多维回测视图：同屏比较 K 线、预测线、净值与收益指标。
- 可解释输出：支持对模型分析结果进行可读化展示与报告下载。
- 配置化后端接入：通过环境变量切换 API 地址，便于本地/测试/生产部署。

## 技术栈

- Vue 3
- TypeScript
- Vite
- Element Plus
- Axios
- ECharts

## 系统结构

- 路由层：统一页面导航与模块入口。
- API 层：按业务域封装请求方法。
- 视图层：实时汇率、预测、回测、用户中心等页面。
- 工具层：请求实例、API 地址拼接、通用工具函数。
- 数据层：`public/data` 提供可用于演示的样例数据。

## 快速开始

### 1. 安装依赖

```bash
npm install
```

### 2. 配置环境变量

复制环境变量模板：

```bash
cp .env.example .env
```

按需修改：

```env
VITE_API_BASE_URL=http://localhost:6020
```

### 3. 启动开发环境

```bash
npm run dev
```

### 4. 构建生产版本

```bash
npm run build
```

### 5. 本地预览构建产物

```bash
npm run preview
```

## 环境变量

- `VITE_API_BASE_URL`：后端服务基础地址（示例：`http://localhost:6020`）。

说明：项目中的 API 请求已统一通过该变量拼接，不再依赖硬编码地址。

## 目录结构（节选）

```text
src/
  api/                # API 封装
  components/         # 通用组件
  router/             # 路由配置
  utils/              # 工具函数（含 API 地址工具）
  views/              # 页面
public/data/          # 回测数据样例
```

## 数据说明

- `public/data/backtest-result/` 与 `public/data/backtest-profit/` 存放回测结果样例数据。
- 若你接入自有后端，可直接替换 `VITE_API_BASE_URL` 指向对应 API 服务。

## 适用场景

- 课程作业与竞赛展示：快速呈现“数据-模型-业务”完整链路。
- 策略研究演示：用于比较不同策略在相同货币对下的表现。
- 前后端联调：作为模型服务或量化 API 的可视化前端。

## 安全与开源说明

- 移除代码中的硬编码后端公网 IP，统一改为环境变量。
- 提供 `.env.example` 作为配置模板。
